**공통 정비
1. 텍스트 정렬
- 모든 텍스트는 왼쪽 정렬 > 왼쪽 정렬시 왼쪽 여백 추가 했으면 좋겠음(의견)
- 수치 데이터는 오른쪽 정렬, 1000 단위는 , > 1,000 확인
- 강조 및 특정 데이터는 가운데 정렬
- 같은 텍스트는 화면 전체에 무조껀 동일하게 표기(띄어쓰기 등 모두 동일하게 표기)

2. 사용유무 활용하기
- 공정코드에서 시유, 포장 콤보박스 가져올 때, 사용유무로 가져오기

3. 안내문구
- 삭제 : "사용되고 있는 데이터는 삭제할 수 없습니다."
- 유효성 체크 : "필수 데이터를 입력하세요" >> 사용자 친화적 방법으로 변경했으면 좋겠음

4. 로드화면
- 모든 데이터 조회 (검색 쿼리는 DB로 직접 가는 형식으로)
- 직접쿼리(ad-hoc)보다는 '저장 프로시저'를 사용 "권장"
- 로드시 데이터 조회 없음(빈화면) > 조회 버튼 누르면 모든 데이터 조회
- 조회시 디폴트로 첫 행 데이터 선택 > 아래 데이터 정보란에 데이터 로드 연계(dgv, row change 이벤트?)
- 추가 또는 수정 완료 했을 때, dgv selected row가 수정 추가된 row로 선택, 해당 데이터 데이터 정보란에 로드 연계

5. 에러 확인
- 콤보박스 텍스트 변화 못하게 하거나, 텍스트 변화에 따른 쿼리 필요
- ucSearch 의경우 검색 조건에서 해당 폼에서 조회되는 데이터는 ucSearch보다 텍스트 박스 형식으로 조회 작성

6. 기타
- '%' + @파라미터 > 파라미터로 끝나는
- @파라미터 '%' > 파라미터로 시작되는
- '%' + @파라미터 + '%' > 파라미터가 포함되는 경우
- 투입실적 > 산출실적 : 중간 로스(떨어짐 등), 산출 실적이 생상량
- 투입main | 투입sub >>>>>>>>>>>>>라인 >>>>>>>>>>>>> 산출 sub | 산출 main 형식
- 간혹 산출이 투입보다 많은 경우 발생
-nop_Type

**화면별 요구 사항
1. 로그인
- 왼쪽 로그인 버튼(clear 기능)이 사용자에게 불친절한 거 같음> 알 수 없음
- 로그인 횟수 제안 적용 여부?

2. 사용자 그룹 관리
- admin 이란 삭제가 불가능한 아이디

3. 인사관리
- 재직여부 : 시스템 코드로 넣는 것을 요구

4. 작업장 정보
- 작업지시 상태 dgv col 삭제 요구, 해당 정보는 작업 지시 상태에 있으면 됨

5. 품목정보
- 반제품인 경우에도 성형이후를 종합하기 때문에 규격이 있는 것을 요구

6. 사용자 정의 코드
- 정렬 순서 int > string으로 요구
(추가, 10,20 ~ / 010, 020 ~ / 100, 200 ~ 처럼 향후 추가 및 수정에 대한 대비 수단으로 정의하는 것을 요구) > 이건 선택 사항

7. 생산 계획 및 요청
- 화면 목적 : 수주 된 내용에 대하여 생산을 요청하는 것이 목적 > 납기일자는 검색 조건은 필요 없어도 괜찮음
- 보통 날짜 조건을 복수로 하지 않음(콤보박스(납기/요청) 으로 선택하는 것이 어떰?
- 납기 경우 납기 현황을 중점으로하는 조회 폼을 별도로 만드는 것을 추천
- 생산계획 생산 요청 DGV 색갈 통일 
- 요청일자 선택이 필요 > 자동저장되면 안됨, / 디폴트 날자를 넣는 방법도 좋음

8. 바코드 생성 및 재발행
- 팔렛 생성 시, 등급을 설정, 재발행 때는 등급은 변경 못함

9. 현장 작업 지시 생성 기능 요구

10. 비가동 발생 규정
자동
- 설비로부터 고장 신호를 받은 경우
- 비가동 자동발생 여부 / 비가동 자동발생시간 : 특정시간동안 실적이 없는 경우, 비가동으로 변환
수동
- 사용자가 직접 비가동 정의
